## ✅ Библиотека - ТЕКСТОВЫЕ данные (реально работает)

### Что сделано:

#### 1. PubMed API интеграция (`server/services/pubmedService.ts`)
- ✅ Поиск статей через E-utilities API
- ✅ JSON формат (не PDF!)
- ✅ ~100KB на статью (вместо 2MB PDF)
- ✅ Экономия 95% места

#### 2. База данных (`server/database/library.sql`)
```sql
- library_articles: 20+ полей (title, abstract, authors, keywords, mesh_terms)
- user_library_items: избранное пользователей
- library_categories: 10 категорий
- researcher_program: программа вознаграждений
- researcher_tasks: задачи для исследователей
- institutional_partners: институциональное сотрудничество
```

#### 3. API endpoints (`server/routes/library.ts`)
- `GET /api/library/categories` - список категорий
- `GET /api/library/articles` - поиск по локальной БД
- `GET /api/library/search-pubmed` - поиск в реальном времени
- `POST /api/library/import-article` - импорт в БД
- `POST /api/library/import-category` - массовый импорт

#### 4. Категории для импорта:
- nutrition (питание)
- fitness (фитнес)
- sleep (сон)
- mental_health (ментальное здоровье)
- weight_loss (похудение)
- cardiovascular (сердечно-сосудистые)
- diabetes (диабет)
- supplements (БАДы)
- meditation (медитация)
- posture (осанка)

### Размеры:
| Формат | Размер на статью | 10,000 статей |
|--------|------------------|---------------|
| PDF | 2 MB | 20 GB |
| JSON (metadata) | 50-100 KB | 1 GB |
| **Экономия** | **95%** | **19 GB** |

## ❌ Что НЕ сделано (и почему):

### 1. Скачивание полных текстов (PDF)
- **Почему нет:** PubMed Central OA - 8M статей = 16 TB
- **Решение:** Храним только metadata, PDF по ссылке

### 2. Полный поисковый движок (Elasticsearch)
- **Почему нет:** Требует отдельный сервер
- **Решение:** PostgreSQL full-text search (достаточно для 50K статей)

### 3. Автоматический импорт всех статей
- **Почему нет:** Rate limits PubMed API (3-10 запросов/сек)
- **Решение:** Фоновый импорт по категориям (1000 статей = ~2 часа)
