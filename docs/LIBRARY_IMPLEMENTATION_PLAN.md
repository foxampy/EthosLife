# План реализации локальной научной библиотеки

## Вопросы для уточнения:

### 1. Масштаб библиотеки
- **Сколько документов** планируется хранить? (100, 1000, 10000+)
- **Какие типы документов?** (PDF статьи, abstracts, метаданные)
- **Размер хранилища** - готовы ли выделить 10GB, 100GB, 1TB?

### 2. Источники данных
Какие базы парсить и скачивать:
- **PubMed Central** (Open Access) - ~8M статей
- **arXiv** - ~2M препринтов
- **bioRxiv/medRxiv** - препринты биологии/медицины
- **DOAJ** (Directory of Open Access Journals)
- **Все сразу** или по категориям?

### 3. Технические детали
- **Хранение**: файловая система или БД (PostgreSQL, MongoDB)?
- **Поиск**: полнотекстовый поиск (Elasticsearch, Typesense)?
- **PDF viewer**: встроенный или скачивание?
- **API**: нужен программный доступ?

### 4. Обновления
- **Как часто** обновлять? (ежедневно, еженедельно)
- **Автоматический парсинг** или ручной импорт?

## Оценка объемов:

| База | Количество | Средний размер | Общий объем |
|------|------------|----------------|-------------|
| PubMed Central OA | 8M статей | 2MB | 16 TB |
| arXiv | 2M статей | 1MB | 2 TB |
| bioRxiv | 100K статей | 2MB | 200 GB |
| **Итого (все)** | **~10M** | - | **~18 TB** |

**Реалистично для старта** (отобранные категории):
- Health/Medicine: ~50K статей = ~100 GB
- Nutrition: ~10K статей = ~20 GB
- Sports: ~10K статей = ~20 GB
- **Итого: ~140 GB**

## Техническая архитектура:

### Вариант 1: Полная локальная библиотека (СЛОЖНО)
```
Backend:
- Python scraper (PubMed E-utilities API)
- PostgreSQL + pgvector (векторный поиск)
- Elasticsearch (полнотекстовый поиск)
- MinIO/S3 (хранение PDF)
- Celery (фоновые задачи парсинга)

Frontend:
- PDF viewer (react-pdf или PDF.js)
- Полнотекстовый поиск
- Фильтры по метаданным
```

**Время разработки**: 2-3 месяца
**Стоимость сервера**: $50-200/мес (зависит от объема)

### Вариант 2: Гибридная модель (ПРОМЕЖУТОЧНЫЙ)
- Хранить только метаданные (авторы, abstract, теги)
- PDF по запросу скачивается/кешируется
- Поиск по метаданным
- Ссылки на оригиналы

**Время разработки**: 2-3 недели
**Стоимость**: $10-20/мес

### Вариант 3: API интеграции (БЫСТРЫЙ)
- PubMed API (E-utilities)
- arXiv API
- Google Scholar API (неофициальный)
- Поиск в реальном времени
- Без хранения PDF

**Время разработки**: 3-5 дней
**Стоимость**: Бесплатно (rate limits)
