## 🔴 КРИТИЧЕСКИЕ ПРОБЛЕМЫ

### 1. ОТСУТСТВУЕТ ПРОДАЮЩАЯ СТРАНИЦА (Landing Page)

**Проблема:**
- Текущая страница `/v2` — это НЕ продающий лендинг
- Нет чёткого оффера "что это и зачем"
- Нет воронки захвата лидов
- Нет социальных доказательств
- Нет чёткого CTA (call-to-action)

**Влияние:**
- Пользователи не понимают ценность продукта
- Низкая конверсия в регистрацию
- Инвесторы не видят масштабируемость

**Решение:**
Создать полноценный landing page (см. `SALES_FUNNEL_DESIGN.md`)

---

### 2. ПУТАНИЦА В НАВИГАЦИИ

**Проблема:**
```
Текущая структура:
- Главная (/) → непонятно что это
- /landing → дубль главной?
- /presentation → ещё одна презентация?
- /v2 → ещё одна версия лендинга?
- /newstyle → ещё один стиль?
```

**Дублирование роутов:**
- `/dashboard` vs `/dashboard-v1` vs `/dashboard-v2`
- `/health-center` vs `/health-center` vs старые модули
- `/specialists` vs `/social/specialists` vs `/specialists-old`

**Влияние:**
- Пользователи теряются
- SEO-проблемы (дубли контента)
- Технический долг

**Решение:**
1. Оставить ОДИН лендинг: `/` (LandingV2 переделать)
2. Один дашборд: `/dashboard`
3. Удалить все `/v2`, `/newstyle`, `/*-old`

---

### 3. НЕТ ДОСТУПА БЕЗ РЕГИСТРАЦИИ

**Проблема:**
- Все модули здоровья требуют входа
- Пользователь не может "пощупать" продукт
- Нет preview режима

**Влияние:**
- 70% посетителей уходят без регистрации
- Нет возможности "попробовать до покупки"

**Решение:**
- Сделать все страницы доступными для просмотра
- Блокировать только персональные данные
- Показывать CTA "Зарегистрируйтесь для сохранения"

---

### 4. МОБИЛЬНАЯ НАВИГАЦИЯ — ПЕРЕГРУЖЕНА

**Текущее состояние:**
```tsx
// BottomNavigation.tsx — 214 строк
// 4 иконки + меню с 20+ пунктами
// Меню открывается слева (неудобно для правшей)
// Многоуровневая структура
```

**Проблемы:**
- 4 иконки внизу — мало для быстрого доступа
- Меню с 20+ пунктами — перегруз
- Нет приоритизации
- Нет "избранного"

**Решение:**
- Tab bar с 5 пунктами (Home, Health, AI, Community, Profile)
- Swipe-навигация между модулями
- Pull-to-refresh

---

### 5. ОТСУТСТВУЕТ ПОСТОЯННЫЙ CTA

**Проблема:**
- На внутренних страницах нет кнопки "Зарегистрироваться"
- Только в хедере (маленькая)
- Нет sticky CTA

**Решение:**
- Sticky banner снизу для незарегистрированных
- Floating action button "Начать"
- Inline CTA в конце каждого раздела

## 🔧 ТЕХНИЧЕСКИЕ РЕКОМЕНДАЦИИ

### 1. Удалить дублирующие файлы
```bash
# Старые лендинги
- client/src/pages/Landing.tsx
- client/src/pages/newstyle/
- client/src/pages/Presentation.tsx

# Старые дашборды
- client/src/pages/Dashboard.tsx

# Старые модули
- client/src/pages/Medicine.tsx (использовать health/MedicineHealth.tsx)
- client/src/pages/Nutrition.tsx
- ... и т.д.
```

### 2. Структурировать роуты
```typescript
// App.tsx — упрощённая структура
<Switch>
  {/* Public */}
  <Route path="/" component={LandingPage} />
  <Route path="/pricing" component={Pricing} />
  <Route path="/health/:module" component={HealthModulePublic} />
  
  {/* Auth */}
  <Route path="/login" component={Login} />
  <Route path="/register" component={Register} />
  
  {/* Protected */}
  <Route path="/dashboard">
    {isAuth ? <Dashboard /> : <Redirect to="/register" />}
  </Route>
</Switch>
```

### 3. Добавить ProtectedRoute компонент
```typescript
// components/ProtectedRoute.tsx
export function ProtectedRoute({ component: Component, ...rest }) {
  const { isAuth } = useAuth();
  const [location] = useLocation();
  
  if (!isAuth) {
    // Сохраняем куда хотел пойти
    sessionStorage.setItem('redirectAfterLogin', location);
    return <Redirect to="/register" />;
  }
  
  return <Component {...rest} />;
}
```
